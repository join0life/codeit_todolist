## 고민한 지점

### TODO/DONE 영역 분리 방식

- 홈 페이지의 'TODO'와 'DONE' 섹션을 슬롯 구조로 분리할지, 혹은 하나의 리스트 컴포넌트에 `isCompleted` 프롭스를 전달해 조건부 렌더링할 지 고민
  - 본 과제에서는 상태 구분이 단순하고, 각 섹션에서 아이템 유무에 따른 빈 화면 UI만 필요했기 때문에 재사용성을 과도하게 분리하기보다 `isCompleted` 값을 기준으로 렌더링을 분기하는 방식 선택

### 아이콘 구현 방식

- Figma에서 제공된 이미지를 png로 추출해 사용할 경우 해상도 저하 문제 발생
  - 최대한 Lucide React 라이브러리를 활용해 아이콘을 구현하고, 피치 못한 경우 Figma 파일에서 SVG를 추출해 이미지 대신 활용

### 상세 페이지 UX

- 사용자 편의를 위해 `isCompleted === false`인 경우, 할 일 제목 `input`에 자동 포커스

## 회고

### 느낀 점

간단한 기능 구현 과제였지만, 구현 과정에서 예상보다 많은 것들을 고민할 수 있는 좋은 시간이었다.
특히, 사용자 경험 관점에서 '이러면 불편할 것 같은데?'하는 지점을 어떻게 개선하면 좋을지 더 공부해야겠다고 느꼈다.

### 아쉬웠던 점

- API 요청 후 로딩 시간이 길게 느껴져 UX가 다소 저하
  - Suspense로 스트리밍을 구현했지만, 추가 개선이 필요하다고 느꼈다.
- 클라이언트 컴포넌트 비중이 높아진 구조
  - 상호작용 중심 UI 특성상 불가피한 부분도 있었지만, 서버 컴포넌트와 클라이언트 컴포넌트 간의 역할 분리를 더 명확하게 가져갈 수 있지 않았을까 하는 아쉬움이 남았다.
