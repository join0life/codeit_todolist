# 코드잇 과제

투두리스트 만들기

## 기술 스택

- Next.js (App Router)
- Typescript

## 페이지 & 기능 구성

- 할 일 목록 페이지(/)
  - 목록 조회
    - [x] 로고 버튼 클릭 시 ('/') 페이지로 이동 (새로고침)
    - [x] 진행 중인 할 일과 완료한 할 일을 나누어 볼 수 있음
  - 할 일 추가
    - [x] 상단 입력창에 할 일 테스트 입력 -> 추가하기 버튼 클릭 & 엔터 -> 할 일 생성
  - 할 일 완료
    - [x] 진행 중 할 일 항목의 왼쪽 버튼을 다시 클릭하면 체크 표시가 사라지고 진행 중 상태가 됨
- 할 일 상세 페이지(/items/{itemsId})
  - 할 일 수정
    - [x] 할 일 항목 클릭한 후 항목 수정 가능
    - [x] 항목 이름 수정
    - [x] 할 일 상태(진행/완료) 수정
    - [x] 메모 추가
    - [x] 이미지 첨부(1개)
      - [x] 페이지 파일 이름은 영어로만
      - [x] 파일 크기는 5MB 이하
    - [x] 수정 완료 버튼 클릭 시 수정 사항 반영, 할 일 목록 페이지로 이동
  - 할 일 삭제
    - [x] 삭제하기 버튼을 클릭하면 할 일 삭제 가능, 할 일 목록 페이지로 이동

## 폴더 구조

```
/src
  /app
    /items
      /[itemId]
        page.tsx           # 할 일 상세 페이지
    /actions
      /todo-actions.ts     # API 호출 함수 + 캐싱
    layout.tsx             # 메인 레이아웃
    page.tsx               # 홈 페이지

  /components              # 공통(재사용) 컴포넌트
  /assets                  # 이미지 파일 폴더


```

## 고민한 것들

- 홈 페이지에서 TODO와 DONE 섹션을 슬롯으로 나눌지, isCompleted 프롭스를 전달해 구분할지 고민
  - 단순히 todo와 done 구분 + 아이템이 없을 때 이미지 UI를 렌더링하므로 isCompleted 프롭스를 전달해 구분하기로 결정
- API 요청 후 Pending
  - 간단히 state로 구현
